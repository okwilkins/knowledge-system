## 18.2 — Introduction to iterators

### Standard library iterators

- Iterating is such a common operation, that all standard library container offer direct support for iteration.
- Instead of manually calculating the begin and end points, the container can be asked for its begin and end points.
	- This is done via member functions named `begin()` and `end()`.


```cpp
#include <array>
#include <iostream>

int main() {
	std::array array{1, 2, 3};
	
	auto begin{array.begin()};
	auto end{array.end()};
	
	for (auto p{begin}; p != end; ++p) {
		std::cout << *p << ' ';	
	}
	
	std::cout << '\n';
	
	return 0;
}
```

- The `iterator` header also contains two generic functions (`std::begin` and `std::end`) that can be used.

```cpp
#include <array>
#include <iostream>

int main() {
	std::array array{1, 2, 3};
	
	auto begin{std::begin(array)};
	auto end{std::end(array)};
	
	for (auto p{begin}; p != end; ++p) {
		std::cout << *p << ' ';
	}
	
	std::cout << '\n';
	
	return 0;
}
```

### Back to range-based for loops

- All types that implement `begin()` and `end()` member functions or can be used with `std::begin()` and `std::end()` are usable in range-based for-loops.

```cpp
#include <array>
#include <iostream>

int main() {
	std::array array{1, 2, 3};
	
	for (int i : array) {
		std::cout << i << ' ';	
	}
	
	std::cout << '\n';
	
	return 0;
}
```

- Behind the scenes, the range-based for-loop calls `begin()` and `end()` of the type to iterator over.
- For example, C-style fixed arrays can be used with `std::begin()` and `std::end()` functions.
	- So this can be used with a range-based for loop.
- Dynamic C-style arrays (or decayed C-style arrays) don't work though.
	- This is because there is no `std::end()` function for them.
		- The type information doesn't contain the array's length.
- Range-based for loops aren't the only thing that makes use of iterators.
	- They are also used in `std::sort` and other algorithms.

### Iterator invalidation (dangling iterators)

- Much like pointers and references, iterators can be left dangling.
	- This happens when the elements being iterates over change address or are destroyed.
	- This is to say that the iterator has been **invalidated**.
- Some operations that modify containers, such as adding an element to a `std::vector`, can have the side effect of causing the elements in the container to change addresses.
	- When this happens, existing iterators to those elements will be invalidated.
- Good C++ reference docs should note which container operations may or will invalidate iterators.

- Since range-based for-loops use iterators behind the scenes, caution must be taken to not do anything that invalidates the iterators of the container that is being actively traversed:
```cpp
#include <vector>

int main() {
	std::vector v{0, 1, 2, 3};
	
	for (auto num : v) {
		if (num % 2 == 0) {
			// When this invalidates the iterators of v,
			// undefined behaviour will result
			v.push_back(num + 1);
		}	
	}
	
	return 0;
}
```

- Here's another example of iterator invalidation:

```cpp
#include <iostream>
#include <vector>

int main() {
	std::vector v{1, 2, 3, 4, 5, 6, 7};
	
	auto it{v.begin()};
	++it;
	std::cout << *it << '\n';

	// Erase the element currently being iterated over
	v.erase(it);
	
	// erase() invalidates iterators to the erased
	// element (and subsequent elements)
	++it;
	// Undefined behaviour
	std::cout << *it << '\n';
	
	return 0;
}
```

- Invalidated iterators can be re-validated by assigning a valid iterator to them (e.g. `begin()`, `end()` or the return valid of some other function that returns an iterator).

- The `erase()` function returns an iterator to the element one past the erased element (or `end()` if the last element was removed).
- Therefore, the above code can be fixed:

```cpp
#include <iostream>
#include <vector>

int main() {
	std::vector v{1, 2, 3, 4, 5, 6, 7};p
	
	auto it{v.begin()};
	++it;
	std::cout << *it << '\n';

	// Erase the element currently being iterated over
	// set it to next element
	it = v.erase(it);
	
	std::cout << *it << '\n';
	
	return 0;
}
```

### Personal Note on std::end

- `std::end` or `container.end()` is NOT the last element.
	- It refers to the past-the-end iterator.
- It is either a `nullptr` (or some other sentinel value) or one element past the last element.
	- One element past the last element will probably point to garbage and should not be attempted to be read, as this will result in undefined behaviour.

## 18.3 — Introduction to standard library algorithms

- Functionality provided in the algorithms library generally falls into one of three categories:

1. **Inspectors**: used to view (but not modify) data in a container. Examples include searching and counting.
2. **Mutators**: used to modify data in a container. Examples include sorting and shuffling.
3. **Facilitators**: used to generate a result based on values of the data members. Examples include objects that multiply values, or objects that determine what order pairs of elements should be sorted in.

### Using std::find to find an element by value

- `std::find` searches for the first occurrence of a value in a container.
- It takes three parameters:
	- An iterator to the starting element in the sequence.
	- An iterator to the ending element in the sequence.
	- A value to search for.
- It returns an iterator pointing to the element (if it is found) or the end of the container (if the element is not found)	.

```cpp
#include <algorithm>
#include <array>
#include <iostream>

int main() {
	std::array{13, 90, 99, 5, 40, 80};
	int search{5};
	int replace{10};
	
	auto found{std::find(arr.begin(), arr.end(), search)};
	
	if (found == arr.end()) {
		std::cout << "Could not find " << search << '\n';	
	} else {
		*found = replace;	
	}
	
	for (int i : arr) {
		std::cout << i << ' ';	
	}
	
	std::cout << '\n';
	
	return 0;
}
```

### Using std::find_if to find an element that matches some condition

- Something seeing if there is a value in a container that matches some condition rather than an exact value.
- In such cases, `std::find_if` is perfect.

- The `std::find_if` fucntion works similarly to `std::find` but instead of passing in a specific value to search for, a callable object, such as a function pointer (or a lambda).
- For each element being iterated over, `std::find_if` will call this function (passing the element as an argument to the function) and the function can return `true` if a match is found, `false` otherwise.

```cpp
#include <algorithm>
#include <array>
#include <iostream>
#include <string_view>

bool containsNut(std::string_view str) {
	// std::string_view::npost is returned from find if
	// a value was not found.
	return str.find("nut") != std::string_view::npos;
}

int main() {
	std::array<std::string_view, 4> arr{"apple", "banana", "walnut", "lemon"};
	
	auto found{std::find_if(arr.begin(), arr.end(), containsNut)};
	
	if (found == arr.end()) {
		std::cout << "No nuts\n";	
	} else {
		std::cout << "Found " << *found << '\n';	
	}
	
	return 0;
}
```

### Using std::count and std::count_if to count how many occurrences there are

```cpp
#include <algorithm>
#include <array>
#include <iostream>
#include <string_view>

bool containsNut(std::string_view str) {
	return str.find("nut") != std::string_view::npos;
}

int main() {
	std::array<std::string_view, 5> arr{"apple", "banana", "walnut", "lemon"};
	
	auto nuts{std::count_if(arr.begin(), arr.end(), containsNut)};
	
	std::cout << "Counted " << nuts << " nut(s)\n";
	
	return 0;
}
```
