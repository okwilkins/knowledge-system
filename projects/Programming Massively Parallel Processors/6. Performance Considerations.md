## 6.1 Global Memory Access Coalescing

 - Memory coalescing techniques are often used in conjunction with tiling techniques.
	 - They allow CUDA devices to reach their performance potential by efficiently utilising the global memory bandwidth.

- The global memory of a CUDA device is implemented with DRAM.
- In DRAM data bits are stored in DRAM cells that are small capacitors.
	- The presence or absence of a tiny amount of electrical charge distinguishes between an 1 or 0.
	- DRAM is Dynamic Random Access Memory.
		- The dynamic part means that it won't hold, data unless refreshed in regular intervals.
- Reading data from a DRAM cell required the small capacitor to use its electrical charge to drive a highly capacitive line leading to a senor and to set off the senor's detection mechanism that determines where a sufficient chart is present in the capacitor to qualify as a 1.
	- This process takes tens of nanoseconds.
	- The clock cycle time of devices are sub-nanosecond.
- Because of this process is very slow, modern DRAM designs use parallelism to increase their rate of data access.
	- Commonly referred to as **memory access throughput**.

- Each time a DRAM location is accessed, a range of consecutive locations that include the requested location are accessed as well.
	- Many sensors are provided in each DRAM chip and they all work in parallel.
	- Each sensor senses the content of a bit within these consecutive locations.
	- The thousands of these cells are connected a horizontal line.
- Once detected by the sensors, the data from all these consecutive location can be transferred at high speed to the processor.
- These consecutive locations accessed and delivered are referred to as **DRAM bursts**.
- If an application makes focused use of data from a burst, the DRAM can supply the data at a much higher than if an application accesses a truly random sequence of locations scattered across many bursts.

- Modern GPUs also use SRAM-based L1 and L2 cache.
	- SRAM is Static Read Only Memory.
	- It will hold its state, for as long as there is power.
- When data in the L2 cache is accessed, a range of consecutive locations are moved to L1 cache from L2.
	- This granularity at which data is moved between different levels of cache is known as the **cache block** or **cache line**.
	- If an application makes focused use of data from within a cache line, the caches can supply the data at a faster rate because fewer cache lines need to be moved.

- When all threads in a warp execute the same load instruction, the hardware detects whether they access consecutive global memory locations.
	- The most favourable access pattern is achieved when all threads in a warp access consecutive global memory locations in the same DRAM burst and/or cache line.
- In this case, the hardware combines or **coalesces** all these accesses into a consolidated access.

- Besides ensuring that threads in a warp access consecutive global memory locations, another consideration is the **alignment** of these accesses in global memory.
	- When the starting address of these consecutive global memory locations is aligned to the beginning of a burst or cache line.
		- This **alignment** ensures that the smallest number of memory transactions are needed to perform the coalesced access.
	- If the starting address is not aligned, the unaligned access could result in additional memory traffic.
		- The accesses data wall span a larger number of bursts or cache lines.

- In linear algebra, it is often needed to use both the original and transposed forms of a matrix.
	- It would be better to avoid creating and storing both forms.
	- A common practice is to create the matrix in one form, say the original form.
	- When the transposed form is needed, its elements can be accessed by accession the original form and switching the roles of the row and column indices.
		 - This is equivalent to viewing the transposed matrix as being in the column-major layout.
		 - This is not favourable for coalescing.

- There are various strategies for optimising code to achieve memory coalescing, when the computation is not naturally amenable to it.
	- Rearrange how threads are mapped to the data.
	- Rearrange the layout of the data itself.
	- Read more in section 6.8.
- Another is to transfer the data from global to shared memory, in a coalesced manner and carry out the unfavourable access pattern in shared memory.

- **Corner turning** is a method where memory from global memory (DRAM) is accessed in a coalesced method and then placed into shared memory in a column-major way in shared memory (SRAM).
	- SMEM is implemented with SRAM and does not require coalescing.
	- SRAM is far more forgiving than DRAM about access orientation.
- In reality, the tile size for corner turning will be the warp size of 32, rather than 4 in the above figure.

- While the example focuses on enabling the coalescing of global memory load operations, coalescing is also important for global memory store operations.
